# Accessibilité (a11y) — Portfolio Callisto Arts

## Réalisé (base)

- Lien d’évitement « Aller au contenu principal », focus sur `<main>`.
- Focus visible au clavier (`:focus-visible`) : navigation, filtres, cartes portfolio, bandeau cookies.
- Textes alternatifs des visuels portfolio / hero / lightbox : fonction `buildArtworkAltText()` dans `script.js` (titre, catégorie, année, indication YouTube si applicable).
- Logos en en-tête / pied de page : `alt=""` + libellé sur le lien parent ou texte à côté (évite la redondance).
- Contrastes : renfort sur textes secondaires (footer, mentions, lecteur audio, `--color-text-tertiary`).

## À valider manuellement

- **Contrastes** : outil (axe DevTools, Lighthouse, WebAIM) sur les combinaisons couleur encore nombreuses (filtres par catégorie, timeline Journey, mode nuit atelier si utilisé).
- **Lecteur audio** : labels et temps déjà renforcés ; revérifier sur fond glass réel.
- **Vidéos YouTube** : titre / contexte dans la lightbox ; iframe avec `title` (déjà présent sur `#lightbox-video`).

## Mobile & tactile

- `env(safe-area-inset-*)` sur la barre de navigation et le menu déroulant ; `scroll-padding-top` pour les ancres sous header fixe.
- Cibles tactiles **≥ 44px** (variable `--touch-target-min`) : bouton menu, liens du menu mobile, filtres portfolio, flèches hero, points du carousel (zone élargie), boutons du bandeau cookies.
- `touch-action: manipulation` sur ces contrôles pour limiter le délai de 300 ms sur certains navigateurs.
