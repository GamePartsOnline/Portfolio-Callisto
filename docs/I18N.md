# i18n (FR + EN) — mode hybride

## Principe

- Le **HTML sert de version française** par défaut : le site reste lisible sans JavaScript.
- Le fichier **`i18n.json`** contient les paires **`fr` / `en`** par clé (chemins pointés par `data-i18n`, ex. `nav.buildLog`).
- Au chargement, le script enregistre une **copie** du HTML français de chaque nœud `[data-i18n]` ; passage en anglais = remplacement par la chaîne **`en`** correspondante ; retour en français = restauration de la copie.
- Les libellés **`aria-label`** traduisibles utilisent **`data-i18n-aria="clé"`** (snapshots séparés).

## Fichiers

| Fichier | Rôle |
|--------|------|
| `i18n.json` | Dictionnaire bilingue (structure imbriquée, clés en chemin pointé) |
| `content.json` | Données de contact (email, téléphone, adresse) — pas de texte d’interface |
| `script.js` | `initI18nApplyStoredLanguage()`, `loadLanguage('fr' \| 'en')` exposé sur `window` |

## Workflow IA (recommandé)

1. Copier le HTML des sections à traduire (ou le texte visible).
2. Demander : *« Transforme ce texte en JSON bilingue structuré pour les clés data-i18n du site (voir i18n.json existant). »*
3. Fusionner les nouvelles clés dans **`i18n.json`** en conservant la même hiérarchie.
4. Ajouter les attributs **`data-i18n="section.sousCle"`** sur les nœuds correspondants dans `index.html` (texte français dans le fichier).
5. Incrémenter **`i18n.json?v=`** dans `script.js` (constante `I18N_JSON_URL`) si le cache pose problème.

## Sélecteur de langue

Barre **`FR | EN`** dans la nav (`class="lang-switch"`). Préférence stockée dans **`localStorage`** sous la clé `callisto-lang`.

## Pages satellites

`build-log.html`, `services.html` et `mentions-legales.html` chargent le même **`script.js`** et **`i18n.json`** que l’accueil : menu, pied de page et sélecteur **FR/EN** sont traduits comme sur `index.html`.

**Journal technique (`build-log.html`)** : l’**en-tête de page** (titre, chapô, ligne « Mis à jour », note sur la langue, pied de note avant contact) est **bilingue** via les clés `buildLogPage.*` dans `i18n.json`. Les **articles** (études de cas) restent rédigés en **anglais** par choix éditorial (preuve de travail, lectorat tech international) ; une **encadré** sous le chapô l’indique explicitement.
